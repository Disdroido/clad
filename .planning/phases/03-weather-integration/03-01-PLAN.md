    **2. Define internal `WeatherApiResponse` interface** matching the WeatherAPI.com response shape (see RESEARCH.md lines 251-275 for the full shape). Keep this interface un-exported — it's internal to the module.

    **3. `fetchCurrentWeather(lat, lon, event?)`** — Core fetch function:
    ```typescript
    import { requireServerEnv } from './runtime-env'
    import type { H3Event } from 'h3'

    export async function fetchCurrentWeather(
      lat: number,
      lon: number,
      event?: H3Event,
    ): Promise<WeatherData | null> {
      try {
        const apiKey = requireServerEnv('weatherApiKey', event)
        const raw = await $fetch<WeatherApiResponse>(
          'https://api.weatherapi.com/v1/current.json',
          { params: { key: apiKey, q: `${lat},${lon}` } },
        )
        return {
          temperature: raw.current.temp_f,
          feelsLike: raw.current.feelslike_f,
          condition: raw.current.condition.text,
          conditionCode: raw.current.condition.code,
          iconUrl: raw.current.condition.icon,
          humidity: raw.current.humidity,
          locationName: raw.location.name,
          lastUpdated: raw.current.last_updated,
        }
      } catch {
        return null  // Graceful degradation (D-05)
      }
    }
    ```

    **4. `fetchWeatherByIp(ipAddress, event?)`** — IP-based fallback for when browser geolocation is unavailable:
    ```typescript
    export async function fetchWeatherByIp(
      ipAddress: string,
      event?: H3Event,
    ): Promise<WeatherData | null> {
      try {
        const apiKey = requireServerEnv('weatherApiKey', event)
        const raw = await $fetch<WeatherApiResponse>(
          'https://api.weatherapi.com/v1/current.json',
          { params: { key: apiKey, q: ipAddress } },
        )
        return {
          temperature: raw.current.temp_f,
          feelsLike: raw.current.feelslike_f,
          condition: raw.current.condition.text,
          conditionCode: raw.current.condition.code,
          iconUrl: raw.current.condition.icon,
          humidity: raw.current.humidity,
          locationName: raw.location.name,
          lastUpdated: raw.current.last_updated,
        }
      } catch {
        return null
      }
    }
    ```

    **Icon URL handling:** The WeatherAPI returns protocol-relative URLs (`//cdn.weatherapi.com/...`). Pass them through as-is — the client can prepend `https:` when rendering (handled in Plan 03-03's WeatherBadge).

    **Why these choices:**
    - No npm dependencies — `$fetch` is built into Nitro/Cloudflare Workers (zero-dep)
    - Null returns on failure for graceful fallback (same try/catch pattern as wear recency D-05)
    - Separate functions for lat/lon vs IP for clarity — both are one-liner wrappers

    ```typescript
    // GET /api/weather/current?lat=37.77&lon=-122.42
    import { fetchCurrentWeather } from '../../utils/weather'

    export default defineEventHandler(async (event) => {
      const query = getQuery(event)
      const lat = parseFloat(query.lat as string)
      const lon = parseFloat(query.lon as string)

      if (isNaN(lat) || isNaN(lon)) {
        return null  // No valid coords — graceful degradation

      return await fetchCurrentWeather(lat, lon, event)
    })
    ```

    **Weather fetch block** — Insert after the wear recency section (after line 71, before `const enrichedProfile`):
    ```typescript
    // 2.7. Fetch weather data (graceful fallback per D-05)
    let weatherResult: WeatherData | null = null
    try {
      const clientIp = getRequestHeader(event, 'x-forwarded-for')?.split(',')[0]?.trim()
      if (body.lat && body.lon) {
        weatherResult = await fetchCurrentWeather(body.lat, body.lon, event)

        // auto:ip on Cloudflare Workers resolves the datacenter, not the user

      }
    } catch {
      // Weather unavailable — engine falls back to climate zone
    }
    ```

    ```typescript
    const enrichedProfile = {
      ...(profile || {}),
      recentlyWornItemIds,

      // ^ use feelsLike for clothing decisions (per research: "feels-like > actual temp")
    }
    ```

    **Update the return statement** (after AI selection, around line 108) to include weather data:
    ```typescript
    return {
      items: candidates[outfitIndex],
      occasion,
      explanation,
      weather: weatherResult ? {
        temperature: weatherResult.temperature,
        condition: weatherResult.condition,
    ```

    **Error handling:** The entire weather fetch is in try/catch. If WeatherAPI is down, the API key is misconfigured, or the quota is exceeded, weatherResult stays null and `currentTemperature` passes `undefined` to the engine. The scoring engine checks `!== undefined` before applying weather scoring (handled in Plan 03-02). This matches the same pattern as wear recency (D-05).

    **Why IP fallback exists:** When the user denies browser geolocation permission, we don't have lat/lon. The client IP from x-forwarded-for gives approximate location. On Cloudflare Workers, this header contains the real visitor IP (set by Cloudflare).
